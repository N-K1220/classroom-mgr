# classroom-mgr

講義室の利用状況を作成・確認・出力する，Ruby 製の対話型 CLI アプリケーション．

以下のデータをもとに，特定年度の各学期において管理対象となる講義室を利用する日ごとの利用状況をまとめた講義室管理情報を作成し，一覧表の Excel ファイルを出力する．

1. 岡山大学授業日程計画の Excel ファイル（学年暦データ）
2. 講義室で実施されるすべての講義科目の時間割情報で構成された Excel ファイル（時間割データ）
3. 講義以外で利用される講義室の事前予約をまとめた Excel ファイル（予約データ）
4. 管理対象とする講義室の情報をまとめた Excel ファイル

## 主な機能

- 学年暦データ，時間割データ，予約データおよび管理対象講義室のデータを Excel ファイルから読み込む
- 学年暦の学期，祝日・休業日，曜日振替，補講日，試験期間を反映して，時間割から授業日を展開する
- 同じ講義室・日時に重複する予定を対話形式で解消する
- 作成した講義室管理情報をターミナル上で検索・表示する
- 学期ごとの講義室利用状況を Excel ファイルとして出力する

## 動作環境

- Ruby 3.2.3
- RubyGems 3.4.19
- Bundler 2.4.19


## セットアップ

gem をインストールする

```bash
bundle install --path vendor/bundle
```

## 入力ファイルの配置

入力データは，アプリケーションルートである`classroom-mgr`配下の`data`に配置する．

### ディレクトリ構成と制約事項
`data`配下に置く入力ディレクトリ名（例: `2026`）は任意の名前でよい．ただし，ファイル名やディレクトリ名にパス区切り文字や`:*?"<>|`などの文字は使用できない．

指定した入力ディレクトリの下に`学年暦`，`時間割`，`予約`というディレクトリを配置し，それぞれの下に Excel ファイルを **1つだけ** 配置する（ファイル名は任意）．

さらに`data`配下へ`管理対象講義室`というディレクトリを作成し，その下に対象講義室の Excel ファイルを **1つだけ** 配置する（ファイル名は任意）．

```text
classroom-mgr/
├── main.rb
├── data/
│   ├── 2026/                     # 入力データを配置するディレクトリ
│   │   ├── 学年暦/
│   │   │   └── calendar.xlsx
│   │   ├── 時間割/
│   │   │   └── timetable.xlsx
│   │   └── 予約/
│   │       └── reservation.xlsx
│   └── 管理対象講義室/
│       └── rooms.xlsx
├── output/                       # 生成される一覧表の出力先
...
```

### 管理対象講義室ファイルの作成
管理対象講義室の Excel ファイルはユーザ自身で作成する．
先頭シートのA列に，管理対象にする講義室名を1行に1つずつ記載する．

| A列 |
| --- |
| 講義室A |
| 講義室B |


## 使い方

以下のコマンドを入力することで，アプリケーションを起動する．

```bash
bundle exec ruby main.rb
```

`>`が表示されたら，以下のコマンドを順に実行する．

```text
> read 2026
> select
> create
> print
> write 2026年度講義室予約表
> quit
```

### 実行時の注意事項
- データはアプリケーションの実行中だけ保持されるため，再起動後は`read`と`select`を再度実行する必要がある．
- `read`で指定できるのは，data直下のディレクトリ名のみである．
- `read`と`select`の実行順序は問わないが，`create`の前に両方を完了させる必要がある．
- `create`の実行後に内容を変更したい場合は，必要なコマンドを改めて実行する必要がある．


### コマンド一覧

| コマンド | 引数 | オプション | 説明 |
| --- | --- | --- | --- |
| `read` | `<入力ディレクトリ名>` | なし | `data/<入力ディレクトリ名>`内の学年暦，時間割，予約を読み込む．例：`read 2026` |
| `select` | なし | なし | `data/管理対象講義室`の Excel を読み込み，表示された講義室を管理対象として確定する．既存の講義室管理情報がある場合はリセットする． |
| `create` | なし | `-t <学期>`，<br>`--term <学期>` | 講義室管理情報を作成する．オプション未指定時は全学期を対象とし，指定時は1〜4の指定学期だけを対象とする（例：`create -t 1`）．講義室に同じ時間の重複予定がある場合は，画面の選択肢から残す予定を選ぶことで，優先する予定として情報を作成する． |
| `print` | なし | `-d <MMDD>`，<br>`--date <MMDD>`，<br>`-s <科目名・予約名>`，<br>`--subject <科目名・予約名>` | 作成済みの講義室管理情報を一覧表示する．`-d`または`--date`で月日を，`-s`または`--subject`で科目名を指定して絞り込める．両方指定した場合は，両方に一致する情報を表示する（例：`print -d 0403 -s "プログラミング演習"`）． |
| `write` | `<ファイル名>` | なし | 利用状況の一覧表を`output`へ出力する．ファイル名が`.xlsx`で終わらない場合，自動で拡張子が付加される．例：`write 2026年度講義室予約表`は`output/2026年度1学期講義室予約表.xlsx`を作成する． |
| `quit` | なし | なし | アプリケーションを終了する． |

## 出力

`write`コマンドを実行すると，`output`ディレクトリに Excel ファイルが作成される．

- 学期ごとのシートが作成され，日付・講義室・時限ごとの利用状況が記載される．
- 各シートには，1限から4限，昼休み，5限から8限の時限と開始・終了時刻が表示される．
- 各日付には，学年暦に基づく備考が併記される．
- 出力対象は，`select`で設定した管理対象講義室に一致する情報のみである．管理対象外の講義室を含む時間割・予約は Excel ファイルに含まれない．
